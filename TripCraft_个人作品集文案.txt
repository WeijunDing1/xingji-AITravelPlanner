一、项目概览

TripCraft 是一个面向自由行用户的 AI 旅行规划助手。

用户只需要输入旅行想法，或上传小红书、抖音等平台的攻略截图，系统就可以自动整理出结构化旅行需求，并生成一份包含日程安排、地图路线、预算明细和可对话修改能力的完整旅行方案。

一句话介绍：
输入旅行想法或攻略截图，自动生成带地图路线、预算明细和 AI 对话修改能力的完整行程。


二、项目背景与问题

很多自由行用户在出发前会收藏大量攻略内容，但这些内容往往是碎片化的：

1. 景点、餐厅、住宿信息分散在不同平台。
2. 用户需要手动判断地点距离和路线是否合理。
3. 行程安排容易出现来回跑、跨区移动过多的问题。
4. 预算通常只能粗略估计，很难拆分到交通、住宿、餐饮和门票。
5. 当用户想调整行程时，传统工具往往需要从头修改。

TripCraft 的目标是把“碎片化攻略”转化为“可执行行程”，降低旅行规划的时间成本。


三、目标用户

目标用户主要是 18-35 岁的自由行用户，包括：

1. 喜欢自己规划路线，但不想花太多时间整理攻略的人。
2. 收藏了大量旅行笔记，却难以整合成完整行程的人。
3. 希望快速获得结构化旅行方案的人。
4. 需要和同行者分享路线、预算与日程的人。


四、核心功能

1. 智能输入

支持自然语言输入旅行需求，例如目的地、天数、预算、人数和偏好。

支持上传攻略截图，AI 自动提取截图中的目的地、景点、餐厅、酒店、亮点和注意事项。

支持快捷标签选择，例如天数、预算、出行人数和兴趣偏好。


2. AI 行程生成

系统根据用户输入生成结构化行程。

行程按天拆分，每天包含景点、餐厅、住宿、推荐时间、停留时长、费用、评分和交通方式。

生成过程中使用进度反馈，让用户知道 AI 当前正在分析需求、搜索信息、规划路线、计算预算或生成方案。


3. 地图路线视图

在地图上标注每天的 POI 点位。

按天切换路线，展示当天游玩路线和地点顺序。

统计当天总距离、步行段数、打车段数等信息，帮助用户判断路线是否合理。


4. 预算明细视图

展示总预算和人均预算。

将费用拆分为交通、住宿、餐饮、门票和其他类别。

支持查看每类费用的具体明细，例如某天餐厅消费、交通费用或住宿费用。


5. AI 对话式修改

结果页提供 AI 悬浮助手。

用户可以用自然语言继续修改行程，例如：

“Day1 晚餐换成本地海鲜店。”
“减少打车，多安排步行。”
“把预算控制在 6000 以内。”

AI 会判断用户意图。如果是咨询问题，就直接回答；如果是修改指令，就只更新相关行程内容，并同步影响日程、地图和预算。


五、用户流程

用户打开首页
↓
输入旅行需求，或上传攻略截图
↓
选择天数、预算、人数和偏好标签
↓
点击“开始规划”
↓
进入 AI 生成进度页
↓
生成完成后进入结果页
↓
用户可以在三个视图之间切换：

1. 日程视图：查看每天的时间线和 POI 卡片。
2. 地图视图：查看路线和地点标注。
3. 预算视图：查看费用构成和明细。

↓
如果不满意，可以通过 AI 悬浮助手继续修改
↓
保存、分享或重新生成行程

七、技术实现

技术栈：

Next.js
React
TypeScript
Tailwind CSS
DeepSeek API
Doubao 多模态 API
高德地图 Web API


实现要点：

1. 使用 Next.js App Router 构建移动端优先的 Web 应用。
2. 使用 React 组件拆分首页、Loading 页、结果页、地图视图、预算视图和 AI 对话组件。
3. 使用 DeepSeek 生成结构化旅行计划 JSON。
4. 使用 Doubao 多模态能力解析攻略截图中的旅行信息。
5. 使用高德地图展示 POI 标记和路线。
6. 使用 SSE 流式返回生成进度，提升等待过程中的反馈感。
7. 使用统一的 TripPlan 数据结构驱动日程、地图和预算三个视图。


八、项目亮点

1. 从碎片攻略到结构化行程

项目不是简单的聊天工具，而是把用户的自然语言和截图内容整理成可执行的旅行计划。


2. 日程、地图、预算三视图联动

同一份行程数据同时驱动日程展示、地图路线和预算拆分，让结果更完整、更适合真实出行场景。


3. 支持截图输入，降低整理成本

很多旅行灵感来自小红书、抖音和攻略截图。通过图片识别，用户不用手动复制景点和餐厅信息。


4. 支持 AI 对话式修改

用户不满意某一部分时，不需要重新生成整份行程，可以直接用自然语言局部修改。


5. 移动端拟 App 体验

界面以手机端为核心，使用卡片、底部操作栏、横向 Day 切换、悬浮 AI 助手等移动端交互方式，体验接近原生 App。

这是我做的一个 AI 旅行规划助手项目，核心目标是把用户收藏的碎片化攻略转成可执行的旅行计划。
