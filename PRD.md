# 行迹 TripCraft — 产品需求文档（PRD）

## 1. 产品概述

### 1.1 产品名称
行迹 TripCraft

### 1.2 一句话定位
AI 旅行规划助手——输入想法或截图，输出带地图路线、预算明细、真实推荐的完整攻略。

### 1.3 目标用户
- 18-35 岁年轻用户，喜欢自由行
- 收藏了大量旅游笔记但不会整理规划的人
- 希望快速获得结构化行程而非碎片化信息的人

### 1.4 核心痛点
| 痛点 | 现状 |
|------|------|
| 信息碎片化 | 收藏了 100 篇攻略，出发前还是一团乱麻 |
| 规划耗时 | 自己排行程要花数小时甚至数天 |
| 路线不合理 | 手动排的行程经常走回头路 |
| 预算模糊 | 不知道整趟下来到底花多少钱 |

### 1.5 产品形态
- Web 应用（响应式移动端 UI 优先）
- 手机浏览器直接访问，体验接近原生 App

## 2. 功能需求

### 2.1 P0 — MVP 必须交付

#### 2.1.1 智能输入
| 功能 | 说明 |
|------|------|
| 文本输入 | 自然语言描述旅行需求（目的地、天数、预算、人数、偏好），最长 500 字 |
| 图片输入 | 上传小红书/抖音旅游攻略截图（最多 8 张），AI 自动提取景点、餐厅、行程信息 |
| 快捷标签 | 可选标签辅助输入：天数（含自定义输入）、预算区间、出行人数、偏好类型 |
| 图文整合 | 图片提取结果自动与文本输入合并，生成综合旅行需求 |

#### 2.1.2 AI 行程生成
| 功能 | 说明 |
|------|------|
| 结构化日程 | 按天拆分，每天包含：景点、餐厅、交通方式、建议时间段 |
| 智能路线 | 按地理分区规划，每天在同一区域内活动（5km 半径），不跨区不走回头路 |
| 预算估算 | 分项列出交通、住宿、餐饮、门票费用，包含每段交通费用 |
| 推荐理由 | 每个 POI 附带推荐原因、评分、人均消费 |

#### 2.1.3 地图可视化
| 功能 | 说明 |
|------|------|
| POI 标注 | 在高德地图上标注所有景点、餐厅位置 |
| 路线绘制 | 绘制每天的游览路线（步行/驾车/公交） |
| 日程切换 | 可按天切换查看不同日期的路线 |

#### 2.1.4 结果展示与操作
| 功能 | 说明 |
|------|------|
| 日程卡片 | 可展开/收起的每日行程卡片 |
| 预算面板 | 饼图或列表展示费用构成，展开可查看各项花销明细（日期、类别、店名） |
| 重新生成 | 不满意可重新生成或微调 |
| 分享 | 生成链接或图片分享给同行人 |

#### 2.1.5 AI 对话助手（追问修改）
| 功能 | 说明 |
|------|------|
| 悬浮球入口 | 结果页可拖拽的 AI 助手悬浮球，松手自动吸附到屏幕边缘 |
| 自然语言修改 | 用户输入修改指令（如"Day1 午餐换成烤鸭"），AI 只改动相关部分 |
| 对话咨询 | 支持纯对话模式，如"推荐几个酒店"直接回答不修改行程 |
| 实时更新 | 修改行程后日程、地图、预算自动同步刷新 |
| 多轮对话 | 保持上下文，支持连续追问修改 |

### 2.2 P1 — 后续迭代

| 功能 | 说明 |
|------|------|
| 行程编辑 | 拖拽调整景点顺序，手动添加/删除 POI |
| 用户登录 | 保存历史行程 |
| 社区广场 | 浏览其他用户的行程，支持"抄作业" |
| 行程 Remix | 基于他人行程一键适配自己的偏好 |
| 实时天气 | 行程中融入天气提醒 |
| 酒店/机票 | 接入预订渠道 |

## 3. 用户流程

```
用户打开首页
    │
    ├─ 选择输入方式
    │   ├─ 文本：描述旅行想法 + 选择快捷标签
    │   └─ 图片：上传截图 → AI 识别内容 → 用户确认/补充
    │
    ├─ 点击 [开始规划]
    │
    ├─ Loading 页面（显示进度："正在分析目的地..."、"规划路线中..."）
    │
    └─ 结果页
        ├─ Tab1: 日程视图（卡片列表）
        ├─ Tab2: 地图视图（路线 + POI）
        ├─ Tab3: 预算视图（分项明细）
        ├─ 底部操作栏：重新生成 / 分享 / 保存
        └─ AI 对话助手（悬浮球）
             └─ 输入修改指令 → AI 局部更新行程 → 页面自动刷新
```

## 4. 业务规则

### 4.1 输入限制
- 文本输入最长 500 字
- 图片支持 JPG/PNG，最大 5MB，最多 8 张
- 行程天数限制：1-30 天（支持自定义输入）

### 4.2 生成规则
- 每天安排 2-4 个景点 + 2-3 餐
- 景点间距合理（单次通勤不超过 1 小时）
- 预算估算基于当地平均消费水平
- 推荐 POI 优先选择评分 4.0 以上
- 所有费用统一使用人民币（海外目的地按汇率换算）
- 交通段（地铁/公交/打车）标注费用
- 人均费用由前端根据出行人数计算（总额 ÷ 人数）

### 4.3 路径优化规则
- 先将目的地所有景点按地理位置分区
- 每天的行程只在一个区域内活动（5km 半径），绝不跨区
- 不同天之间不重复去同一区域
- 每天路线为单向链路（A→B→C→D），无回头路
- 多城市行程按城市分天，不同一天跨城
- 住宿安排在当天最后一个景点附近

### 4.4 AI 对话规则
- AI 自动判断用户意图：咨询（纯对话）vs 修改（更新行程）
- 纯咨询：直接回答，不修改行程 JSON
- 修改指令：只改相关部分，自动重算预算
- 支持多轮上下文对话

## 5. 非功能需求

| 维度 | 要求 |
|------|------|
| 性能 | 行程生成 < 15 秒 |
| 兼容性 | iOS Safari / Android Chrome / 微信内置浏览器 |
| 并发 | MVP 阶段支持 50 并发用户 |
| 安全 | 不存储用户上传图片（处理后即删） |

## 6. 成功指标（MVP 阶段）

| 指标 | 目标 |
|------|------|
| 行程生成成功率 | > 95% |
| 平均生成时间 | < 12 秒 |
| 用户单次使用时长 | > 2 分钟 |
| 分享率 | > 15% |
